# Mental-Health-Support-System-Database-Project
A web-based platform designed to provide a structured and accessible environment for emotional well-being tracking, personal journaling, and professional therapy connections.

## 📖 Project Overview
In today’s fast-paced world, mental health is a critical concern. This system integrates modern web technologies with mental health awareness to create a safe space for users to monitor their moods, seek professional guidance, and receive intelligent recommendations for their well-being.

## ✨ Key Functionalities
* **User Management:** Secure login and role-based access for Patients, Therapists, and Admins.
* **Mood Tracking:** Log daily moods with intensity levels (1-10) and view weekly/monthly trends.
* **Journal Management:** A secure, private space to document personal thoughts and review past entries.
* **Session Booking:** Browse specialized therapists and book, cancel, or reschedule appointments.
* **Resource Sharing:** Access a library of articles, guides, and videos curated by professionals.
* **Intelligent Recommendation Engine:** Automatically detects patterns of negative moods and suggests relevant resources or sessions.
* **Advanced Analytics:** Summarized mood patterns and therapist activity reports using SQL aggregations.

## 🛠️ Tech Stack
### Frontend
* **React.js:** For building a modular and responsive UI.
* **HTML5 & CSS3:** Semantic structure and clean, device-agnostic styling.
* **JavaScript (ES6):** Dynamic interactivity and API communication.

### Backend & Database
* **Oracle Database:** Main storage for user accounts, logs, and resources.
* **SQL/PLSQL:** Implementation of Triggers, Stored Procedures, Joins, and Views for complex operations.
* **PHP / Node.js:** Server-side logic connecting the frontend to the Oracle backend.
* **Oracle SQL Developer:** Used for database design, testing, and debugging.

## 📊 Database Operations
The system utilizes advanced database concepts including:
* **CRUD Operations:** Standard Create, Read, Update, and Delete functionality.
* **Triggers & Stored Procedures:** Specifically for the recommendation engine and automated reporting.
* **Complex Queries:** Utilization of `GROUP BY`, `JOINS`, and Aggregations for user analytics.
